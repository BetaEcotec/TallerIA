POWERSHELL · 5 MIN

Preparar el entorno

Abre PowerShell como Administrador (clic derecho → “Ejecutar como administrador”) y pega estos comandos uno por uno.

PowerShell — instalar Node.js
# Instala Node.js (incluye npm) con el gestor de paquetes de Windows
winget install OpenJS.NodeJS.LTS -e

# Cierra y vuelve a abrir PowerShell. Luego verifica que quedó instalado:
node -v
npm -v
⚠️ Si winget no se reconoce, actualiza la “App Installer” desde la Microsoft Store, o descarga Node.js desde nodejs.org (versión LTS).
2
POWERSHELL · BASE DE DATOS

Instalar una instancia de MongoDB

La base de datos es donde el servidor guarda cada alerta. Instalaremos MongoDB, que se ejecuta como un servicio (una “instancia” siempre encendida) en tu computadora.

PowerShell — instalar MongoDB
# Instala el servidor de MongoDB (queda como servicio en localhost:27017)
winget install MongoDB.Server -e

# (Opcional) Instala Compass, una interfaz gráfica para ver los datos
winget install MongoDB.Compass.Full -e

# Verifica que el servicio de la base de datos está corriendo:
Get-Service MongoDB
PowerShell — si el servicio no está iniciado
# Inícialo manualmente (requiere PowerShell como Administrador)
Start-Service MongoDB
¿Prefieres usar Docker en vez de instalar MongoDB? (alternativa)

Docker te deja levantar la base de datos como un “contenedor” — otra forma de instancia. Útil si no quieres instalar MongoDB directamente.

# 1) Instala Docker Desktop (pedirá reiniciar la primera vez)
winget install Docker.DockerDesktop -e

# 2) Con Docker abierto, levanta una instancia de MongoDB en un comando:
docker run -d --name alertas-db -p 27017:27017 mongo:7
🧠 Concepto clave: una “instancia” de base de datos es simplemente un motor de base de datos en ejecución, escuchando en un puerto (aquí el 27017) para que tu servidor se conecte a él.
✅ No te preocupes si algo falla aquí: el servidor del Paso 4 tiene un respaldo en memoria y funcionará igual mientras solucionas la base de datos.

4
EL SERVIDOR · NODE + EXPRESS

Crear el servidor que conecta todo

El servidor es el intermediario: recibe las alertas de la app y las reenvía al dashboard. Crea la carpeta del proyecto e instala las dependencias:

PowerShell — crear el proyecto
# Crea la carpeta del taller y entra en ella
mkdir taller-alertas; cd taller-alertas

# Inicializa el proyecto de Node e instala las librerías
npm init -y
npm install express cors mongodb

# Crea la carpeta donde vivirán la app y el dashboard
mkdir publico

Ahora crea un archivo llamado server.js dentro de taller-alertas y pega este código:

server.js
// server.js — Servidor de alertas (Node.js + Express + MongoDB)
const express = require('express');
const cors = require('cors');
const path = require('path');
const { MongoClient } = require('mongodb');

const app = express();
const PUERTO = 3000;
const URL_MONGO = 'mongodb://127.0.0.1:27017';

app.use(cors());
app.use(express.json());
app.use(express.static(path.join(__dirname, 'publico'))); // sirve app.html y dashboard.html

let coleccion = null;   // colección de MongoDB (si está disponible)
let memoria = [];       // respaldo si no hay base de datos
const dashboards = [];  // conexiones del dashboard en tiempo real

// Intenta conectar a MongoDB; si no, usa memoria temporal
(async () => {
  try {
    const cliente = new MongoClient(URL_MONGO, { serverSelectionTimeoutMS: 1500 });
    await cliente.connect();
    coleccion = cliente.db('taller').collection('alertas');
    console.log('✅ Conectado a MongoDB');
  } catch (e) {
    console.log('⚠️  MongoDB no disponible: usando memoria temporal');
  }
})();

// 1) Recibe una alerta desde la app del teléfono
app.post('/api/alertas', async (req, res) => {
  const alerta = {
    tipo: req.body.tipo || 'baja',
    mensaje: req.body.mensaje || 'Alerta sin descripción',
    origen: req.body.origen || 'App móvil',
    fecha: new Date().toISOString()
  };
  if (coleccion) await coleccion.insertOne(alerta);
  else memoria.unshift(alerta);
  // Avisa a todos los dashboards conectados, al instante
  dashboards.forEach(d => d.write('data: ' + JSON.stringify(alerta) + '\n\n'));
  res.json({ ok: true, alerta });
});

// 2) Lista las alertas (cuando el dashboard carga)
app.get('/api/alertas', async (req, res) => {
  const datos = coleccion
    ? await coleccion.find().sort({ fecha: -1 }).limit(50).toArray()
    : memoria;
  res.json(datos);
});

// 3) Canal en tiempo real (Server-Sent Events)
app.get('/api/stream', (req, res) => {
  res.set({ 'Content-Type': 'text/event-stream', 'Cache-Control': 'no-cache', Connection: 'keep-alive' });
  res.flushHeaders();
  dashboards.push(res);
  req.on('close', () => {
    const i = dashboards.indexOf(res);
    if (i >= 0) dashboards.splice(i, 1);
  });
});

app.listen(PUERTO, () => console.log('🚀 Servidor en http://localhost:' + PUERTO));
¿Qué hace cada ruta?
• POST /api/alertas → la app envía aquí la alerta.
• GET /api/alertas → el dashboard pide el historial.
• GET /api/stream → el dashboard queda “escuchando” para recibir alertas nuevas al instante.

LA APP MÓVIL

La pantalla que emite alertas

Dentro de la carpeta publico, crea app.html con este contenido. Es la app del teléfono que envía la alerta al servidor.

publico/app.html
<!DOCTYPE html>
<html lang="es">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>App de Alertas</title>
  <style>
    body{font-family:sans-serif;background:#0a0f24;color:#fff;text-align:center;padding:24px}
    h1{color:#46d6e6}
    button{display:block;width:100%;padding:16px;margin:10px 0;border:none;border-radius:12px;
      font-size:16px;font-weight:bold;cursor:pointer;color:#0a0f24}
    .alta{background:#ff6b6b}.media{background:#f5b841}.baja{background:#5ef2a0}
    #estado{margin-top:16px;color:#9b7bf0}
  </style>
</head>
<body>
  <h1>🚨 Emisor de Alertas</h1>
  <p>Toca un botón para enviar una alerta al dashboard.</p>
  <button class="alta"  onclick="enviar('alta','Emergencia crítica')">🔴 Alerta ALTA</button>
  <button class="media" onclick="enviar('media','Situación a revisar')">🟡 Alerta MEDIA</button>
  <button class="baja"  onclick="enviar('baja','Aviso informativo')">🟢 Alerta BAJA</button>
  <p id="estado"></p>
  <script>
    async function enviar(tipo, mensaje){
      const estado = document.getElementById('estado');
      try{
        const r = await fetch('/api/alertas', {
          method:'POST',
          headers:{'Content-Type':'application/json'},
          body: JSON.stringify({ tipo, mensaje, origen:'App móvil' })
        });
        const data = await r.json();
        estado.textContent = '✅ Enviada: ' + data.alerta.tipo;
      }catch(e){
        estado.textContent = '❌ Error: ¿está corriendo el servidor?';
      }
    }
  </script>
</body>
</html>

EL DASHBOARD

La pantalla que recibe las alertas

Ahora crea dashboard.html, también dentro de publico. Carga el historial y escucha las alertas nuevas en tiempo real.

publico/dashboard.html
<!DOCTYPE html>
<html lang="es">
<head>
  <meta charset="UTF-8">
  <title>Dashboard de Alertas</title>
  <style>
    body{font-family:sans-serif;background:#0a0f24;color:#fff;padding:24px}
    h1{color:#f5b841}
    .alerta{border-left:6px solid #46d6e6;background:#151b38;padding:12px 16px;margin:8px 0;border-radius:8px}
    .alta{border-color:#ff6b6b}.media{border-color:#f5b841}.baja{border-color:#5ef2a0}
    .fecha{font-size:12px;color:#9b7bf0;margin-top:4px}
  </style>
</head>
<body>
  <h1>📊 Dashboard de Alertas (Back-office)</h1>
  <p>Las alertas aparecen aquí en tiempo real.</p>
  <div id="lista"></div>
  <script>
    const lista = document.getElementById('lista');
    function pintar(a){
      const div = document.createElement('div');
      div.className = 'alerta ' + a.tipo;
      div.innerHTML = '<strong>' + a.tipo.toUpperCase() + '</strong> — ' + a.mensaje +
        '<div class="fecha">' + new Date(a.fecha).toLocaleString() + ' · ' + a.origen + '</div>';
      lista.prepend(div);
    }
    // 1) Carga las alertas ya guardadas
    fetch('/api/alertas').then(r => r.json()).then(d => d.reverse().forEach(pintar));
    // 2) Escucha nuevas alertas en tiempo real
    const stream = new EventSource('/api/stream');
    stream.onmessage = (e) => pintar(JSON.parse(e.data));
  </script>
</body>
</html>

¡A PROBAR!

Encender el servidor y ver la magia

Desde la carpeta taller-alertas en PowerShell, enciende el servidor:

PowerShell — arrancar el servidor
node server.js

Verás 🚀 Servidor en http://localhost:3000. Ahora abre dos pestañas en tu navegador:

  • 📱 La app: http://localhost:3000/app.html
  • 📊 El dashboard: http://localhost:3000/dashboard.html

Coloca las ventanas lado a lado, toca un botón en la app… y observa cómo la alerta aparece al instante en el dashboard. 🎉

🛠️ Solución de problemas frecuentes
  • “node no se reconoce” → cierra y abre PowerShell otra vez; si sigue, reinstala Node.js.
  • El dashboard no muestra nada → confirma que el servidor esté corriendo y que abriste las URLs con http://localhost:3000 (no abriendo el archivo directamente).
  • “Error: ¿está corriendo el servidor?” en la app → vuelve a la pestaña de PowerShell; el servidor debe seguir encendido.
  • Puerto 3000 ocupado → cambia PUERTO = 3000 por 3001 en server.js.
  • MongoDB no conecta → no pasa nada: el servidor usa memoria temporal. Revisa Get-Service MongoDB.

🚀 Retos para seguir aprendiendo

  • Sonido y color: haz que el dashboard emita un “bip” y parpadee cuando llega una alerta ALTA.
  • Filtro: agrega botones en el dashboard para mostrar solo alertas de cierta prioridad.
  • Ubicación: que la app envíe también la geolocalización del teléfono con la alerta.
  • Persistencia real: confirma con MongoDB Compass que las alertas quedan guardadas aunque reinicies el servidor.
  • Publicar: vuelve a Google AI Studio y usa Deploy (Cloud Run) para poner tu app en internet.
